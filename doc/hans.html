2. 简介

Ogmacam系列相机支持多种API, 包括：Native C/C++, .NET(C#和VB.NET), Python, Java, DirectShow, Twain, LabView, MatLab等等. Native C/C++ API作为底层(Low Level) API相比较其他API的特点是使用纯C/C++开发, 不依赖其他的运行时库, 接口简洁, 控制灵活. 本SDK压缩包包含了所有需要用到的的资源和信息, 目录如下:

3. 概念和术语

a. 获取图像数据的模式: “Pull” Mode vs “Push” Mode (“拉”模式 vs “推”模式)

Ogmacam提供了两种模式来获取图像数据: Pull Mode 和 Push Mode. 以下所讲的几种方式中, 从功能上说它们都是平等的, 开发者可以任选一种使用. 推荐使用拉模式, 因为它更简单,且在多线程情况下更加不容易出错, 尤其是使用windows消息机制的情况下.

  • Pull Mode, ogmacam 扮演被动角色, 应用程序从ogmacam“拉”图像数据. ogmacam内部线程从相机获得图像数据,并保存到内部缓存中, 然后通知应用程序来取图像数据. 应用程序可以调用Ogmacam_PullImageV3或Ogmacam_PullImage(WithRowPitch)(V2), Ogmacam_PullStillImage(WithRowPitch)(V2)来获取图像数据. 存在两种方式来通知应用程序:
    a) 使用Windows消息机制: 通过调用函数 Ogmacam_StartPullModeWithWndMsg启动Pull mode模式. 当事件发生时, ogmacam会主动发送消息(PostMessage)到指定窗口. 参数WPARAM 是事件类型, 请参考OGMACAM_EVENT_xxxx的定义. 参数LPARAM保留未使用. 本模式规避了多线程问题, 是最简单的方式.(显然, 这种方式只支持Windows系统, 不支持Linux和macOS.)
    b) 使用回调函数使用 Ogmacam_StartPullModeWithCallback启动Pull mode模式. 当事件发生时, 会调用POGMACAM_EVENT_CALLBACK回调函数.

在Pull Mode 情况下, ogmacam不但可以通知应用程序图像数据或者静态图片到达, 还可以通知其他事件类型, 如下所示:

b. 静态抓拍(静态图片, Still Image, Snap)

大部分的相机型号都支持所谓静态抓拍的能力, 指相机在连续的视频预览过程中, 临时切换到另外一个分辨率, 抓取一帧静态图片之后, 马上把分辨率切换回原始分辨率的过程.

举例来说, UCMOS05100KPA支持3种分辨率, 假设当前视频预览分辨率为1280 * 960, 调用函数Ogmacam_Snap(h, 0)静态抓拍分辨率2592 * 1944的静态图片, 这时相机临时切换到2592 * 1944的分辨率, 抓取一帧数据之后, 又把分辨率切换成原来的1280 * 960.

a) 拉模式下, 抓拍到静态图片之后, 通知外层应用OGMACAM_EVENT_STILLIMAGE事件, 然后外层应用调用Ogmacam_PullStillImage(V2)获取静态图片的数据.
b) 推模式下, 抓拍到静态图片之后, 回调函数POGMACAM_DATA_CALLBACK_V4/V3, 参数bSnap设置为TRUE, 图片的分辨率等信息在参数pHeader中.

可以通过函数Ogmacam_get_StillResolutionNumber的返回值或者结构OgmacamModelV2的still值来查看是否支持静态抓拍能力.

c. 数据格式: RGB vs RAW

Ogmacam支持两种数据格式: RGB格式(默认)和RAW格式. RAW模式可以通过调用函数Ogmacam_put_Option设置参数OGMACAM_OPTION_RAW为1开启.

  • RGB格式: 本模式下, 获取底层传感器的数据之后, 进行内部的颜色处理, 输出结果中每像素包含3个分量分别代表R/G/B值.
  • RAW格式: 本模式下, 直接获取底层传感器的“裸”数据, 不进行内部颜色处理. 具体的像素格式(Pixel Format)可以调用函数Ogmacam_get_RawFormat获取. RAW格式模式下, 有关颜色处理的功能(如白平衡)和函数（如Ogmacam_put_Hue,Ogmacam_AwbOnce等等）都无意义.

用户可以通过OGMACAM_OPTION_RAW调用函数Ogmacam_put_Option来切换这两种模式. 请注意相机必须处于非运行状态才能修改此设置.

用户可以通过OGMACAM_OPTION_RGB调用函数Ogmacam_put_Option来设置RGB的bits数. 请注意相机必须处于非运行状态才能修改此设置.

d. 白平衡和自动白平衡: Temp/Tint模式 vs RGB Gain模式

1. Ogmacam支持互相独立的两种模式描述白平衡: a) Temp/Tint模式; b) RGB Gain模式

a) 默认是Temp/Tint模式, 在本模式下, 使用Temp, Tint这2个参数来控制白平衡. Ogmacam_get_TempTint获取值, Ogmacam_put_TempTint设置值. Ogmacam_AwbOnce执行自动白平衡. 当白平衡参数改变时, 发送OGMACAM_EVENT_TEMPTINT通知消息.

b) 在RGB Gain模式下, 使用3个通道的Gain值来控制白平衡. Ogmacam_get_WhiteBalanceGain获取值, Ogmacam_put_WhiteBalanceGain设置值. Ogmacam_AwbInit执行自动白平衡. 当白平衡参数改变时, 发送OGMACAM_EVENT_WBGAIN通知消息.

两种模式下使用的函数不能混淆:

a) Temp/Tint模式下, 必须使用Ogmacam_get_TempTint和Ogmacam_put_TempTint和Ogmacam_AwbOnce. 而Ogmacam_get_WhiteBalanceGain和Ogmacam_put_WhiteBalanceGain和Ogmacam_AwbInit不能使用, 永远返回E_NOTIMPL.
b) RGB Gain模式下, 必须使用Ogmacam_get_WhiteBalanceGain和Ogmacam_put_WhiteBalanceGain和Ogmacam_AwbInit. 而Ogmacam_get_TempTint和Ogmacam_put_TempTint和Ogmacam_AwbOnce不能使用, 永远返回E_NOTIMPL

两种模式在相机打开的时候指定, 除非关闭相机重新打开, 一旦指定就不能更改. 参阅这里.

2. 自动白平衡功能, 业界有两种模式, 一种是连续自动白平衡,一种是触发式自动白平衡(once). 连续自动白平衡功能会一直进行白平衡参数的计算, 触发模式只是在触发的时候才会计算白平衡参数. Ogmacam使用触发式白平衡计算方法, 因为这种方法在显微镜等领域更加合适和精确. 连续自动白平衡在某些场景情况下会出现错误.

3. 黑白相机不支持白平衡. 以上提到的函数一直返回E_NOTIMPL.

e. 触发模式

1. 什么是触发

Ogmacam相机拥有2中工作模式: 视频模式和触发模式. 当处于触发模式时, 只有触发条件满足的情况下才能获取图像. 根据触发源有2种类型的触发类型: 外部触发, 软件触发.

2. 触发和Snap（静态图像抓拍）的区别

触发模式被设计用来精确控制相机当且仅当触发条件满足时才能获取图像. 用户可以通过控制预设的触发条件获取图像. 当进入触发模式时, 直到触发条件被满足才能得到图像. 图像的分辨率没有改变. Snap(静态图像抓拍)是用来在视频模式下抓取相同或者不同分辨率的图像.

3. 软件触发

相机可以被软件触发. 在软件触发模式下, 通过软件来控制触发条件. 图像的张数也可以软件控制.

4. 外部触发

相机可以被外部信号触发. 当前仅支持上升沿触发模式.

5. 混合触发

外部触发+软件触发同时启用.

6. 模拟触发

对于既不支持软件触发又不支持外部触发的相机, 可以使用模拟触发. 当处于模拟触发模式时, 相机硬件的工作模式和视频模式下的工作模式没有区别, 但是上层软件不发起抓取动作, 软件内部的缓冲区保持为空, 直到用户设置触发条件.

7. 怎样进入触发模式

枚举相机时, 可以得到相机型号的能力标志位, 查看相机对于触发模式的支持能力, 定义如下: 
#define OGMACAM_FLAG_TRIGGER_SOFTWARE   0x00080000  /* 支持软触发 */
#define OGMACAM_FLAG_TRIGGER_EXTERNAL   0x00100000  /* 支持外部触发 */
#define OGMACAM_FLAG_TRIGGER_SINGLE     0x00200000  /* 仅支持单张触发: 一次触发得到一张图像 */
可以使用函数Ogmacam_put_Option(HOgmacam h, unsigned iOption, int iValue)设置相机的触发模式, 其中iOption参数为OGMACAM_OPTION_TRIGGER, iValue用于设置触发模式的类型. 请参阅以下内容: 
#define OGMACAM_OPTION_TRIGGER   0x0b    /* 0 = 视频模式, 1 = 软件或模拟触发模式, 2 = 外部触发模式, 3 = 外部触发+软件触发模式；默认值 = 0 */
函数Ogmacam_get_Option(HOgmacam h, unsigned iOption, int* piValue)可以用来获取当前相机的触发模式类型.

8. 怎样触发相机

使用函数Ogmacam_Trigger(HOgmacam h, unsigned short nNumber). 不同的nNumber的含义:
nNumber = 0 取消触发.
nNumber = 0xFFFF 一直触发, 类似于视频模式;
nNumber = 其他合法值代表单次触发获取的图片张数.
如果OGMACAM_FLAG_TRIGGER_SINGLE标志位是设置的, 那么nNumber参数只能为1, 意味着单次触发只能获取单张图片.
在调用Ogmacam_Trigger函数之前, 相机必须已经处于触发模式.

9. 触发超时

超时时间建议不少于(曝光时间 * 102% + 4秒).

  • 回调函数: POGMACAM_EVENT_CALLBACK和POGMACAM_DATA_CALLBACK_V4/V3

    这些回调函数是从ogmacam.dll的内部线程上下文中回调出来, 所以, 非常有必要关注多线程问题. 请尽量保持回调函数代码的简洁, 并且快速返回.
    回调模式下, 如果POGMACAM_EVENT_CALLBACK回调函数复杂, 可以设置OGMACAM_OPTION_CALLBACK_THREAD, 使用专门的线程用于回调.

    由于执行以下操作时必须等待回调函数返回, 回调函数上下文有以下限制:
    (a)不要在回调函数上下文调用Ogmacam_Stop或Ogmacam_Close函数, 否则, 会死锁.

    (c)不要在回调函数上下文调用Ogmacam_put_Roi, 否则返回E_WRONG_THREAD.
    (d)如果执行以上操作时和回调函数时试图获取同一把锁, 会永远导致等待回调结束, 死锁

5. .NET(C#和VB.NET)

Ogmacam支持.NET开发环境(C#和VB.NET).

ogmacam.cs使用P/Invoke调用至ogmacam.dll. 把ogmacam.cs拷贝到你的C#工程中使用, 请参考示例代码.

请主意Ogmacam C#包装类的实例通过静态方法Open或者OpenByIndex得到, 而不是通常的obj = new Ogmacam(构造函数特意是私有的).

绝大多数Ogmacam类的方法和属性都P/Invoke调用至ogmacam.dll/so中对应的原生C/C++ Ogmacam_xxx函数. 所以, 关于Ogmacam_xxx的文档说明都适用于C#中对应的属性或方法, 比如, C#中的Snap方法调用Ogmacam_Snap函数, 关于Ogmacam_Snap函数的说明同样适用于C# Ogmacam类的Snap方法：

[DllImport("ogmacam.dll", ExactSpelling = true, CallingConvention = CallingConvention.StdCall)]
private static extern int Ogmacam_Snap(SafeHOgmacamHandle h, uint nResolutionIndex);

public bool Snap(uint nResolutionIndex)
{
    if (_handle == null || _handle.IsInvalid || _handle.IsClosed)
        return false;
    return (Ogmacam_Snap(_handle, nResolutionIndex) >= 0);
}

VB.NET和C#是类似的, 不另说明.

6. Python

Ogmacam支持Python(版本3.0及以上)开发, 请import ogmacam导入ogmacam.py, 参考示例代码simplest.py和qt.py.

请注意ogmacam.Ogmacam python包装类的实例通过类方法(classmethod)Open或者OpenByIndex得到, 而不是通常的obj = ogmacam.Ogmacam().

绝大多数Ogmacam类的方法都通过ctypes调用至ogmacam.dll/so/dylib中对应的原生C/C++ Ogmacam_xxx函数. 所以, 关于Ogmacam_xxx的文档说明都适用于python中的对应方法.

参见ogmacam.py中的__errcheck, 原始的HRESULT返回值被映射成HRESULTException异常(win32平台从OSError继承).

ogmacam dll/so/dylib库文件请和ogmacam.py存放在同一目录.
